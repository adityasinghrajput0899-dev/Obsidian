**User:** who handles normalization and reconciliation

**Assistant:** Based on the provided information, I will provide a detailed answer.

**Normalization:**

The normalization process is handled by the CSS (Content Services and Standards) team, specifically the Normalization subsystem. The primary point of contact (POC) for normalization is fsainula.

Here's a step-by-step explanation of the normalization process:

1. The incoming payload is normalized based on Enumeration rules/mappings for structured attributes.
2. Derivation Logic derives attribute values from unstructured attribute data.
3. The normalized data is then published to the catalog.

The normalization system has different rules in the DNS system depending on MP-PT-Att. The data comes to the DNS system through sellers/vendors/feeds (contribution data) via the Product KIS Workflow (this workflow is changing).

**Reconciliation:**

The reconciliation process is handled by the SM (Supply Management) team, specifically the External Product Data (EPC) subsystem. The primary point of contact (POC) for reconciliation is Principal PM: ramsvk@.

Here's a step-by-step explanation of the reconciliation process:

1. The system handles selection data from external sources and processes it through various stages before integration into the main pipeline.
2. The system uses the normalized data from the CSS team and then uses this data to reconcile with external data sources.
3. The reconciled data is then sent to the item service.

**Contact Information:**
