**User:** normalization

**Assistant:** Normalization is a crucial process in our cataloging system. According to the Publishing SOP (Enumeration_Discovery.Publishing_SOP.WebHome) - XWiki.md, normalization means taking different inputs that mean the same thing and standardizing them to one canonical (official) value.

In the context of our system, normalization is used to standardize attribute values in a standardized, acceptable format that can be published to the catalog. The normalization system serves two key functions:

1. **Structured Attributes**: The incoming payload is normalized based on Enumeration rules/mappings.
2. **Derivation Logic**: Derives attribute values from unstructured attribute data.

The system maintains synonyms for enum values and normalizes these values to the canonical enum value on the ASIN (Amazon Standard Identification Number). However, if a synonym represents an existing picker value but is added as a synonym for an enum value, it may impact assignments to that picker. This is known as an ASIN drop.

For example, if the system normalizes the term "IR" to "Wireless", the picker might no longer find the ASINs it used to, because it's now looking for a value that no longer exists under that name.

Normalization mappings can be found in the Starfish Modular Title Quality Audits, which provides canonical representations for terms like "blk" and "GPS". For instance:

* "Blk" → "Black"
* "Gps" → "GPS"

These mappings are used to ensure consistency and accuracy in our cataloging system.
