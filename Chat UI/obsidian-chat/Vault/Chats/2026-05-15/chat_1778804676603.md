* The system replaces connectionType = IR with connectionType = Wireless
* The system has different rules in DNS system depending on MP-PT-Att for the normalization of attribute values.
* The system also has Derivation Logic - Derives attribute values from unstructured attribute data.

**Catalog System Map** 📌 Source: [Catalog System Map _ System Details.md]

* The Normalization system's primary purpose is to normalise the attribute values in a standardised, acceptable valid values that can be published to catalog.
* It serves 2 key functions:
  1. For structured attributes, the incoming payload is normalised based on Enumeration rules/mappings.
  2. Derivation Logic - Derives attribute values from unstructured attribute data.
